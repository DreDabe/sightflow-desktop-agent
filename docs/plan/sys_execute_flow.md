# 系统执行流程
1. 布局检测：未读区域、微信布局（解析搜索框、名称/标题区域、对话主窗口，再反推输入框位置）
2. 对象路由：截图获取对话对象名称进行路由。
3. 文本提取：条件步骤，即条件满足则提取聊天文本。
4. 情感分析：条件步骤，即模式若是选择则经过情感分析，若是没有选择则不经过。
5. 调用模型：将聊天界面截图和一些可选内容（情感分析结果、提取的聊天文本），调用模型针对输入给出回复。
6. 推荐回复：系统获取到模型的回复，输出到推荐回复文本框中，等待用户操作。
7. 未读检测：像素diff查看对话窗口是否有变化、模型检测红点/未读区域查看是否有新的未读信息，有未读信息则模拟点击打开聊天框。
8. 等待重试：若是没有新的消息则每等待5秒重新尝试。
